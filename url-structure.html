URL Structure

URL Structure is one of the most important parts of a website, as it is influential in getting pages indexed in search engines, as well being sharable by users.

The best way to make sure a URL is accessible is to create static URLs. This URL structure does not change dependent on the content being served, and the SEO benefit is that static URLs can be read by Google’s crawlers and indexed for inclusion in the search results.

Best practices:

  • Always use semantic URLs over non-semantic
  • Optimize the query string for the page’s target keyword
  • Ideally, this should also replicate the naming convention for the page that’s used in internal linking, to help users identify where they are on-site
  • Use ‘-‘ to separate keywords and avoid underscores.
  • Avoid uppercase: Different servers treat uppercase characters in different ways. In some cases, duplicate content issues can be caused. Although not a hugely important factor, it is good practice to keep all URLs lowercase as they are more user-friendly.
  • URLs should be keyword rich
  • Generally shorter URLs perform better (maximum of 5 relevant words).
  • Describe subfolders and any pages they contain

Example: http://www.threemediacentre.co.uk/Press-Releases/Three-reports-continued-growth-in-profit-and-innovation-370.aspx (Good static structure but too long, with 9 words separated by ‘-‘.

Meta Description

Meta descriptions sit in the <head> of a page’s HTML and are the 928 pixel width visible piece of text below the page titles in search results.

Different to page titles, they are no longer used as a ranking signal. Copy in the meta descriptions should be used to boost click through rate, but remain accurate as to reflect the content on the page, otherwise search engines may deem it too irrelevant and display text generated from the webpage instead.

  1. Ensure that each page has a unique description
  2. Descriptions don’t count towards Google’s ranking algorithm, but do directly impact on the likelihood of users clicking through to the page from search results
  3. Google bolds search queries, synonyms and co-occurring keywords when they appear in descriptions, making it prudent to optimize them
  4. The maximum description width is approximately 920 pixels (for desktop), beyond which it will be truncated or rewritten
  5. This equates to around 120-145 keywords, depending on which letters are used
  6. An example optimized description is: Looking for a dongle, tablet or mobile Wi-Fi connection? Join the UK's fastest growing network and get online wherever you are.

Parameters

There are occasions when content will need to be served dynamically with the use of parameters. A parameter is identified by the ‘?’ seen in the URL. This is a great way of delivering dynamic content and can work for SEO if handled correctly.

The suggested way to handle parameters is through Webmaster tools services for search engines. These can be used to tell search engines which parameters you have on site, which will then allow them to create and store a snapshot of the dynamic page, which can then be crawled, indexed and potentially rank in search results.
